Punto 1

¿Que es el Poliformismo?

El polimorfismo en programación es un concepto que se refiere a la capacidad de un objeto para tomar varias formas o comportarse de diferentes maneras según el contexto en el que se utiliza. Esto significa que un mismo método o función puede comportarse de manera distinta dependiendo del tipo de objeto al que se aplica.

Punto 2

Tipos de Poloformismo

  • Polimorfismo de sobrecarga (Overloading):
    Se refiere a tener múltiples funciones o métodos con el mismo nombre pero con diferentes parámetros. Esto permite llamar a una función utilizando diferentes argumentos y que el compilador seleccione automáticamente la versión correcta según los tipos de datos proporcionados.

  • Polimorfismo de sobreescritura (Override):
    Este tipo de polimorfismo se aplica en la programación orientada a objetos. Ocurre cuando una clase hija redefine un método que ha heredado de su clase padre. La nueva implementación en la clase hija reemplaza la implementación original de la clase padre para ese método.

  • Polimorfismo paramétrico:
    También conocido como polimorfismo genérico, es una característica que permite la creación de clases o métodos que pueden trabajar con distintos tipos de datos sin especificar estos tipos de manera explícita. Esto se logra utilizando parámetros genéricos que se sustituyen por tipos específicos en tiempo de compilación.

  • Polimorfismo ad hoc (ad hoc polymorphism):
    Este tipo de polimorfismo se refiere a la sobrecarga de operadores y funciones, donde el comportamiento de una función u operador varía según el tipo de datos con el que se utiliza. Por ejemplo, en algunos lenguajes, el operador + puede ser utilizado para sumar números enteros, flotantes o concatenar cadenas de texto.

  • Polimorfismo de inclusión (Inclusion polymorphism):
    Es la capacidad de tratar objetos de clases derivadas como objetos de su clase base durante la ejecución del programa. Esto se logra a través de la herencia y el uso de punteros o referencias a la clase base para acceder a los métodos y propiedades de las clases derivadas.

    ¿Que es la sobreescritura de metodos?

    La sobreescritura de métodos es un concepto fundamental en la programación orientada a objetos que permite a una clase hija proporcionar una implementación específica para un método que ha heredado de su clase padre. Esto significa que la clase hija puede redefinir (o sobrescribir) el comportamiento de un método de la clase padre para adaptarlo a sus propias necesidades.

    Cuando una clase hija sobrescribe un método de la clase padre, la firma del método (nombre, tipo de retorno y lista de parámetros) debe ser la misma que la del método original en la clase padre. La clase hija puede modificar el comportamiento del método o agregar funcionalidad adicional, pero debe mantener la misma firma para garantizar la coherencia y compatibilidad con el polimorfismo.

    ¿Que beneficios ofrece el polimorfismo a la POO?

  • Reutilización del código:
    Permite reutilizar métodos y funciones al tratar objetos de diferentes clases de manera uniforme. Esto reduce la duplicación de código y facilita el mantenimiento.

  • Flexibilidad y extensibilidad:
    Al utilizar el polimorfismo, el código se vuelve más flexible y extensible. Se pueden agregar nuevas clases y métodos sin necesidad de cambiar el código existente, siempre y cuando sigan las mismas interfaces o herencias.

  • Facilita la implementación de interfaces y abstracciones:
    El polimorfismo permite trabajar con interfaces y abstracciones, lo que mejora la modularidad y la estructura del código. Por ejemplo, se pueden definir interfaces genéricas y luego implementarlas de manera específica en varias clases.

  • Encapsulamiento y ocultamiento de información:
    Al tratar los objetos a través de sus interfaces y comportamientos comunes, el polimorfismo ayuda a mantener el principio de encapsulamiento, ya que oculta la implementación específica de las clases y solo expone las funcionalidades necesarias.

  • Facilita el uso de herencia y relaciones de clases:
    El polimorfismo es esencial en la jerarquía de clases y la herencia, ya que permite que las clases hijas proporcionen implementaciones específicas para los métodos heredados de las clases padres.

  • Código más legible y mantenible:
    Al utilizar polimorfismo, el código tiende a ser más claro y legible, ya que se pueden tratar objetos de diferentes tipos de manera coherente y comprensible. Esto facilita el mantenimiento y la colaboración entre desarrolladores.

    Tres ejemplos de polimorfismo

  • Polimorfismo de sobreescritura (override):
    Un ejemplo típico es el método Draw en una jerarquía de clases relacionadas con figuras geométricas. Cada clase (como Circle, Rectangle, Triangle) podría sobrescribir el método Draw para dibujar la forma correspondiente de manera específica.

  • Polimorfismo de sobrecarga (overloading):
    En un sistema de gestión de archivos, se podría tener un método Copy sobrecargado que acepte diferentes tipos de parámetros, como Copy(string sourcePath, string destinationPath) para copiar archivos, Copy(Directory sourceDirectory, Directory destinationDirectory) para copiar directorios, y así sucesivamente.

  • Polimorfismo de inclusión (Inclusion polymorphism):
    En un programa de gestión de recursos humanos, podrías tener una lista de empleados de diferentes tipos (como EmpleadoBase, EmpleadoTemporario, EmpleadoContratado). Al tratar a todos estos empleados como objetos de la clase base Empleado, se puede utilizar polimorfismo para acceder a métodos comunes como CalcularSalario(), pero cada tipo de empleado podría tener su propia implementación de este método.
